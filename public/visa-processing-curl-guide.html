Visa Processing API cURL Guide

This file is the working guide for the current visa-processing routes in this Laravel project. Every example below is designed to be copy-paste friendly for terminal use or for giving directly to Codex.

Base URL in examples: http://127.0.0.1:8000/api. Replace it if your app runs on a different host.

Recommended Order

  • Login as admin and create country, visa type, and document requirements.
  • Login as user and browse public country/type/requirement endpoints.
  • Create visa application draft.
  • Add applicant info.
  • Upload documents.
  • Submit application.
  • Pay visa fee.
  • Admin assigns, verifies documents, updates status, and prints PDF.

Shell Variables

export BASE_URL="http://127.0.0.1:8000/api"
export USER_TOKEN="paste_user_token_here"
export ADMIN_TOKEN="paste_admin_token_here"
export COUNTRY_ID=1
export VISA_TYPE_ID=1
export VISA_REQUIREMENT_ID=1
export VISA_APPLICATION_ID=1
export VISA_DOCUMENT_ID=1
export OFFICER_ID=2
export BOOKING_ID=1
export PACKAGE_BOOKING_ID=1

Login First

Both user and admin visa endpoints require Sanctum bearer tokens. Use your existing /api/login route to get them.

cURL
curl --request POST "$BASE_URL/login" \
  --header "Accept: application/json" \
  --header "Content-Type: application/json" \
  --data '{
    "email": "admin@example.com",
    "password": "secret123"
  }'
Sample Response
{
  "access_token": "1|exampleSanctumToken",
  "token_type": "Bearer",
  "expires_in": null,
  "user": {
    "id": 1,
    "name": "Admin User",
    "email": "admin@example.com",
    "role": "admin"
  }
}
Use an admin token for all /admin/visa/* endpoints and a user token for all user visa actions.

Public Visa APIs

GET /countries

Public GET

Get all active visa countries. Optional query: search.

cURL
curl --request GET "$BASE_URL/countries?search=mal" \
  --header "Accept: application/json"

User Visa APIs

User Token Required POST
Requiredcountry_id, visa_type_id
Optionalbooking_id, package_booking_id, remarks

  --header "Accept: application/json" \

POST /visa/pay

User Token Required POST

Pay the visa fee. A valid booking reference may be required because the existing payments table depends on booking_id.

cURL
curl --request POST "$BASE_URL/visa/pay" \
  --header "Accept: application/json" \
  --header "Authorization: Bearer $USER_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "visa_application_id": 5,
    "amount": 5000,
    "payment_method": "bkash",
    "booking_id": 1,
    "bkash": "01700000000"
  }'
Sample Response
{
  "isExecute": "SUCCESS",
  "data": {
    "visa_payment_id": 3,
    "payment_id": 27,
    "transaction_reference": "AB12CD34EF",
    "booking_id": 1,

GET /admin/visa/print/{id}

Admin Token Required GET

This route downloads a PDF, not JSON.

cURL
curl --request GET "$BASE_URL/admin/visa/print/$VISA_APPLICATION_ID" \
  --header "Authorization: Bearer $ADMIN_TOKEN" \
  --output "visa-application-$VISA_APPLICATION_ID.pdf"
Expected Result
A PDF file named visa-application-5.pdf is downloaded successfully.

How To Use This With Codex

Use public/visa-processing-curl-guide.html as the source of truth.
Run the requests in the "Recommended Order" section.
Before each request:
1. Replace BASE_URL and token placeholders.
2. Save returned IDs for country, visa type, requirement, application, and document.
3. Use those IDs in the next request.
4. Stop on any FAILED response and inspect message/errors.
5. For file upload requests, replace the sample file path with a real local file.
